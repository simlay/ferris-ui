12/// A reference counted pointer type for Objective-C objects.
13///
14/// [`Retained`] strongly references or "retains" the given object `T`, and
15/// decrements the retain count or "releases" it again when dropped, thereby
16/// ensuring it will be deallocated at the right time.
17///
18/// The type `T` inside `Retained<T>` can be anything that implements
19/// [`Message`].
20///
21/// This can usually be gotten from one of the methods in [the framework
22/// crates], but can also be created manually with the [`msg_send!`] macro, or
23/// even more manually with the [`Retained::retain`], [`Retained::from_raw`]
24/// and [`Retained::retain_autoreleased`] methods.
25///
26/// [the framework crates]: crate::topics::frameworks_list
27/// [`msg_send!`]: crate::msg_send
28///
29///
30/// # Comparison to `std` types
31///
32/// `Retained<T>` is the Objective-C equivalent of [`Arc`], that is, it is a
33/// thread-safe reference-counting pointer, and allows cloning by bumping the
34/// reference count, and weak references using [`rc::Weak`].
35///
36/// Unlike `Arc`, objects can be retained directly from a `&T` using
37/// [`Message::retain`] (for `Arc` you need `&Arc<T>`).
38///
39/// Even though most Objective-C types aren't thread safe, Objective-C has no
40/// concept of [`Rc`]. Retain/release operations are always atomic.
41///
42/// [`Arc`]: alloc::sync::Arc
43/// [`rc::Weak`]: crate::rc::Weak
44/// [`Rc`]: std::rc::Rc
45///
46///
47/// # Forwarding implementations
48///
49/// Since `Retained<T>` is a smart pointer, it [`Deref`]s to `T`.
50///
51/// It also forwards the implementation of a bunch of standard library traits
52/// such as [`PartialEq`], [`AsRef`], and so on, so that it becomes possible
53/// to use e.g. `Retained<NSString>` as if it was `NSString`. Note that having
54/// `NSString` directly is not possible since Objective-C objects cannot live
55/// on the stack, but instead must reside on the heap, and as such must be
56/// accessed behind a pointer or a reference (i.e. `&NSString`).
57///
58/// Note that because of current limitations in the Rust trait system, some
59/// traits like [`Default`], [`IntoIterator`], [`FromIterator`], [`From`] and
60/// [`Into`] are not directly implementable on `NSString`; for that use-case,
61/// we instead provide the [`DefaultRetained`], [`RetainedIntoIterator`] and
62/// [`RetainedFromIterator`] traits, which make some of the the aforementioned
63/// traits implementable on `Retained`.
64///
65/// [`DefaultRetained`]: crate::rc::DefaultRetained
66/// [`RetainedIntoIterator`]: crate::rc::RetainedIntoIterator
67/// [`RetainedFromIterator`]: crate::rc::RetainedFromIterator
68///
69///
70/// # Memory layout
71///
72/// This is guaranteed to have the same size and alignment as a pointer to the
73/// object, `*const T`.
74///
75/// Additionally, it participates in the null-pointer optimization, that is,
76/// `Option<Retained<T>>` is guaranteed to have the same size as
77/// `Retained<T>`.
78///
79///
80/// # Example
81///
82/// Various usage of `Retained` on an immutable object.
83///
84/// ```
85/// # use objc2::runtime::NSObject;
86/// # objc2::extern_class!(
87/// #     #[unsafe(super(NSObject))]
88/// #     struct NSString;
89/// # );
90/// # #[cfg(available_in_foundation)]
91/// use objc2_foundation::{NSObject, NSString};
92/// use objc2::rc::Retained;
93/// use objc2::{ClassType, msg_send};
94///
95/// // Use `msg_send!` to create an `Retained` with correct memory management
96/// //
97/// // SAFETY: The types are correct, and it is safe to call the `new`
98/// // selector on `NSString`.
99/// let string: Retained<NSString> = unsafe { msg_send![NSString::class(), new] };
100/// // Or:
101/// // let string = NSString::new();
102///
103/// // Methods on `NSString` is usable via `Deref`
104/// # #[cfg(available_in_foundation)]
105/// assert_eq!(string.length(), 0);
106///
107/// // Bump the reference count of the object.
108/// let another_ref: Retained<NSString> = string.clone();
109///
110/// // Convert one of the references to a reference to `NSObject` instead
111/// let obj: Retained<NSObject> = string.into_super();
112///
113/// // And use the `Debug` impl from that
114/// assert_eq!(format!("{obj:?}"), "");
115///
116/// // Finally, the `Retained`s go out of scope, the reference counts are
117/// // decreased, and the string will deallocate
118/// ```
119#[repr(transparent)]
120#[doc(alias = "id")]
121#[doc(alias = "Id")] // Previous name
122#[doc(alias = "StrongPtr")]

129    /// A pointer to the contained object. The pointer is always retained.
130    ///
131    /// It is important that this is `NonNull`, since we want to dereference
132    /// it later, and be able to use the null-pointer optimization.
133    ///
134    /// Additionally, covariance is correct because we're either the unique
135    /// owner of `T`, or `T` is immutable.
136    ptr: NonNull<T>,
137    /// Necessary for dropck even though we never actually run T's destructor,
138    /// because it might have a `dealloc` that assumes that contained
139    /// references outlive the type.
140    ///
141    /// See <https://doc.rust-lang.org/nightly/nomicon/phantom-data.html>
142    item: PhantomData<T>,
143    /// Marks the type as !UnwindSafe. Later on we'll re-enable this.
144    ///
145    /// See <https://github.com/rust-lang/rust/issues/93367> for why this is
146    /// required.
147    notunwindsafe: PhantomData<&'static mut ()>,
148}

167impl<T: ?Sized + Message> Retained<T> {
168    /// Construct an [`Retained`] from a pointer that already has +1 retain count.
169    ///
170    /// Returns `None` if the pointer was NULL.
171    ///
172    /// This is useful when you have a retain count that has been handed off
173    /// from somewhere else, usually Objective-C methods like `init`, `alloc`,
174    /// `new`, `copy`, or methods with the `ns_returns_retained` attribute.
175    ///
176    /// If you do not have +1 retain count, such as if your object was
177    /// retrieved from other methods than the ones noted above, use
178    /// [`Retained::retain`] instead.
179    ///
180    ///
181    /// # Safety
182    ///
183    /// You must uphold the same requirements as described in [`Retained::retain`].
184    ///
185    /// Additionally, you must ensure the given object pointer has +1 retain
186    /// count.
187    ///
188    ///
189    /// # Example
190    ///
191    /// Comparing different ways of creating a new `NSObject`.
192    ///
193    /// ```
194    /// use objc2::rc::Retained;
195    /// use objc2::runtime::NSObject;
196    /// use objc2::{msg_send, AnyThread, ClassType};
197    ///
198    /// // Manually using `msg_send!`, pointers and `Retained::from_raw`
199    /// let obj: *mut NSObject = unsafe { msg_send![NSObject::class(), alloc] };
200    /// let obj: *mut NSObject = unsafe { msg_send![obj, init] };
201    /// // SAFETY: `-[NSObject init]` returns +1 retain count
202    /// let obj: Retained<NSObject> = unsafe { Retained::from_raw(obj).unwrap() };
203    ///
204    /// // Or automatically by specifying `Retained` as the return value from
205    /// // `msg_send!` (it will do the correct conversion internally).
206    /// let obj: Retained<NSObject> = unsafe { msg_send![NSObject::alloc(), init] };
207    ///
208    /// // Or using the `NSObject::new` method
209    /// let obj = NSObject::new();
210    /// ```
211    #[inline]
212    // Note: We don't take a reference as a parameter since it would be too
213    // easy to accidentally create two aliasing mutable references.
214    pub unsafe fn from_raw(ptr: *mut T) -> Option<Self> {
215        // Should optimize down to a noop.
216        // SAFETY: Upheld by the caller
217        NonNull::new(ptr).map(|ptr| unsafe { Retained::new_nonnull(ptr) })
218    }
219
220    /// Deprecated alias for [`Retained::from_raw`], see that for details.
221    ///
222    ///
223    /// # Safety
224    ///
225    /// Same as [`Retained::from_raw`].
226    #[deprecated = "use the more descriptive name `Retained::from_raw` instead"]
227    #[inline]
228    pub unsafe fn new(ptr: *mut T) -> Option<Self> {
229        // SAFETY: Upheld by caller
230        unsafe { Self::from_raw(ptr) }
231    }
232
233    /// Consumes the `Retained`, returning a raw pointer with +1 retain count.
234    ///
235    /// After calling this function, the caller is responsible for the memory
236    /// previously managed by the `Retained`.
237    ///
238    /// This is effectively the opposite of [`Retained::from_raw`], see that for
239    /// more details on when this function is useful.
240    ///
241    ///
242    /// # Examples
243    ///
244    /// Converting an `Retained` to a pointer and back.
245    ///
246    /// ```
247    /// use objc2::rc::Retained;
248    /// use objc2::runtime::NSObject;
249    ///
250    /// let obj = NSObject::new();
251    /// let ptr = Retained::into_raw(obj);
252    /// // SAFETY: The pointer is valid, and has +1 retain count from above.
253    /// let obj = unsafe { Retained::from_raw(ptr) }.unwrap();
254    /// ```
255    #[inline]
256    pub fn into_raw(this: Self) -> *mut T {
257        ManuallyDrop::new(this).ptr.as_ptr()
258    }
259
260    /// Returns a raw pointer to the object.
261    ///
262    /// The pointer is valid for at least as long as the `Retained` is held.
263    ///
264    /// This is an associated method, and must be called as `Retained::as_ptr(obj)`.
265    #[inline]
266    pub fn as_ptr(this: &Self) -> *const T {
267        this.ptr.as_ptr()
268    }
269
270    #[inline]
271    pub(crate) fn as_nonnull_ptr(&self) -> NonNull<T> {
272        self.ptr
273    }
274
275    #[inline]
276    pub(crate) fn consume_as_ptr_option(this: Option<Self>) -> *mut T
277    where
278        T: Sized,
279    {
280        this.map(|this| Retained::into_raw(this))
281            .unwrap_or_else(ptr::null_mut)
282    }
283}

287    /// Attempt to downcast the object to a class of type `U`.
288    ///
289    /// See [`AnyObject::downcast_ref`] for more details.
290    ///
291    /// # Errors
292    ///
293    /// If casting failed, this will return the object back as the [`Err`]
294    /// type. If you do not care about this, and just want an [`Option`], use
295    /// `.downcast().ok()`.
296    ///
297    /// # Example
298    ///
299    /// Cast a string to an object, and back again.
300    ///
301    /// ```
302    /// # use objc2::runtime::NSObject;
303    /// # objc2::extern_class!(
304    /// #     #[unsafe(super(NSObject))]
305    /// #     #[derive(Debug)]
306    /// #     struct NSString;
307    /// # );
308    /// # impl NSString {
309    /// #     objc2::extern_methods!(
310    /// #         #[unsafe(method(new))]
311    /// #         fn new() -> objc2::rc::Retained<Self>;
312    /// #     );
313    /// # }
314    /// # #[cfg(available_in_foundation)]
315    /// use objc2_foundation::{NSString, NSObject};
316    ///
317    /// let string = NSString::new();
318    /// // The string is an object
319    /// let obj = string.downcast::<NSObject>().unwrap();
320    /// // And it is also a string
321    /// let string = obj.downcast::<NSString>().unwrap();
322    /// ```
323    ///
324    /// Try to cast an object to a string, which will fail and return the
325    /// object in [`Err`].
326    ///
327    /// ```
328    /// # use objc2::runtime::NSObject;
329    /// # objc2::extern_class!(
330    /// #     #[unsafe(super(NSObject))]
331    /// #     #[derive(Debug)]
332    /// #     struct NSString;
333    /// # );
334    /// # #[cfg(available_in_foundation)]
335    /// use objc2_foundation::{NSString, NSObject};
336    ///
337    /// let obj = NSObject::new();
338    /// let obj = obj.downcast::<NSString>().unwrap_err();
339    /// ```
340    //
341    // NOTE: This is _not_ an associated method, since we want it to be easy
342    // to call, and it does not conflict with `AnyObject::downcast_ref`.
343    #[inline]
344    pub fn downcast<U: DowncastTarget>(self) -> Result<Retained<U>, Retained<T>>
345    where
346        Self: 'static,
347    {
348        let ptr: *const AnyObject = Self::as_ptr(&self).cast();
349        // SAFETY: All objects are valid to re-interpret as `AnyObject`, even
350        // if the object has a lifetime (which it does not in our case).
351        let obj: &AnyObject = unsafe { &*ptr };
352
353        if obj.is_kind_of_class(U::class()).as_bool() {
354            // SAFETY: Just checked that the object is a class of type `U`,
355            // and `T` is `'static`.
356            //
357            // Generic `U` like `NSArray<NSString>` are ruled out by
358            // `U: DowncastTarget`.
359            Ok(unsafe { Self::cast_unchecked::<U>(self) })
360        } else {
361            Err(self)
362        }
363    }

365    /// Convert the type of the given object to another.
366    ///
367    /// This is equivalent to a `cast` between two pointers.
368    ///
369    /// See [`Retained::into_super`], [`ProtocolObject::from_retained`] and
370    /// [`Retained::downcast`] for safe alternatives.
371    ///
372    /// This is common to do when you know that an object is a subclass of
373    /// a specific class (e.g. casting an instance of `NSString` to `NSObject`
374    /// is safe because `NSString` is a subclass of `NSObject`), but do not
375    /// want to pay the (very slight) performance price of dynamically
376    /// checking that precondition with a [`downcast`].
377    ///
378    /// All `'static` objects can safely be cast to [`AnyObject`], since that
379    /// assumes no specific class.
380    ///
381    /// This is an associated method, and must be called as
382    /// `Retained::cast_unchecked(obj)`.
383    ///
384    /// [`AnyObject`]: crate::runtime::AnyObject
385    /// [`ProtocolObject::from_retained`]: crate::runtime::ProtocolObject::from_retained
386    /// [`downcast`]: Self::downcast
387    ///
388    ///
389    /// # Safety
390    ///
391    /// You must ensure that the object can be reinterpreted as the given
392    /// type.
393    ///
394    /// If `T` is not `'static`, you must ensure that `U` ensures that the
395    /// data contained by `T` is kept alive for as long as `U` lives.
396    ///
397    /// Additionally, you must ensure that any safety invariants that the new
398    /// type has are upheld.
399    #[inline]
400    pub unsafe fn cast_unchecked<U: Message>(this: Self) -> Retained<U> {
401        let ptr = ManuallyDrop::new(this).ptr.cast();
402        // SAFETY: The object is forgotten, so we have +1 retain count.
403        //
404        // Caller verifies that the returned object is of the correct type.
405        unsafe { Retained::new_nonnull(ptr) }
406    }
407
408    /// Deprecated alias of [`Retained::cast_unchecked`].
409    ///
410    /// # Safety
411    ///
412    /// See [`Retained::cast_unchecked`].
413    #[inline]
414    #[deprecated = "Use `downcast`, or `cast_unchecked` instead"]
415    pub unsafe fn cast<U: Message>(this: Self) -> Retained<U> {
416        unsafe { Self::cast_unchecked(this) }
417    }

419    /// Retain the pointer and construct an [`Retained`] from it.
420    ///
421    /// Returns `None` if the pointer was NULL.
422    ///
423    /// This is useful when you have been given a pointer to an object from
424    /// some API, and you would like to ensure that the object stays around
425    /// while you work on it.
426    ///
427    /// For normal Objective-C methods, you may want to use
428    /// [`Retained::retain_autoreleased`] instead, as that is usually more
429    /// performant.
430    ///
431    /// See also [`Message::retain`] for a safe alternative where you already
432    /// have a reference to the object.
433    ///
434    ///
435    /// # Safety
436    ///
437    /// The pointer must be valid as a reference (aligned, dereferenceable and
438    /// initialized, see the [`std::ptr`] module for more information), or
439    /// NULL.
440    ///
441    /// You must ensure that any data that `T` may reference lives for at
442    /// least as long as `T`.
443    ///
444    /// [`std::ptr`]: core::ptr
445    #[doc(alias = "objc_retain")]
446    #[inline]
447    pub unsafe fn retain(ptr: *mut T) -> Option<Retained<T>> {
448        // SAFETY: The caller upholds that the pointer is valid or null
449        let res: *mut T = unsafe { objc_retain_fast(ptr.cast()) }.cast();
450        debug_assert_eq!(res, ptr, "objc_retain did not return the same pointer");
451        // SAFETY: We just retained the object, so it has +1 retain count
452        unsafe { Self::from_raw(res) }
453    }

455    /// Retains a previously autoreleased object pointer.
456    ///
457    /// This is useful when calling Objective-C methods that return
458    /// autoreleased objects, see [Cocoa's Memory Management Policy][mmRules].
459    ///
460    /// This has exactly the same semantics as [`Retained::retain`], except it can
461    /// sometimes avoid putting the object into the autorelease pool, possibly
462    /// yielding increased speed and reducing memory pressure.
463    ///
464    /// Note: This relies heavily on being inlined right after [`msg_send!`],
465    /// be careful to not accidentally require instructions between these.
466    ///
467    /// [mmRules]: https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/MemoryMgmt/Articles/mmRules.html
468    /// [`msg_send!`]: crate::msg_send
469    ///
470    ///
471    /// # Safety
472    ///
473    /// Same as [`Retained::retain`].
474    #[doc(alias = "objc_retainAutoreleasedReturnValue")]
475    #[inline]
476    pub unsafe fn retain_autoreleased(ptr: *mut T) -> Option<Retained<T>> {
477        // Add magic nop instruction to participate in the fast autorelease
478        // scheme.
479        //
480        // See `callerAcceptsOptimizedReturn` in `objc-object.h`:
481        // https://github.com/apple-oss-distributions/objc4/blob/objc4-838/runtime/objc-object.h#L1209-L1377
482        //
483        // We will unconditionally emit these instructions, even if they end
484        // up being unused (for example because we're unlucky with inlining,
485        // some other work is done between the objc_msgSend and this, or the
486        // runtime version is too old to support it).
487        //
488        // It may seem like there should be a better way to do this, but
489        // emitting raw assembly is exactly what Clang and Swift does:
490        // swiftc: https://github.com/apple/swift/blob/swift-5.5.3-RELEASE/lib/IRGen/GenObjC.cpp#L148-L173
491        // Clang: https://github.com/llvm/llvm-project/blob/889317d47b7f046cf0e68746da8f7f264582fb5b/clang/lib/CodeGen/CGObjC.cpp#L2339-L2373
492        //
493        // Note that LLVM may sometimes insert extra instructions between the
494        // assembly and the `objc_retainAutoreleasedReturnValue` call,
495        // especially when doing tail calls and it needs to clean up the
496        // function frame. Unsure how to avoid this in a performant manner?
497        // Maybe force not doing tail calls by inserting assembly to do the
498        // call manually?
499        //
500        // Resources:
501        // - https://www.mikeash.com/pyblog/friday-qa-2011-09-30-automatic-reference-counting.html
502        // - https://www.galloway.me.uk/2012/02/how-does-objc_retainautoreleasedreturnvalue-work/
503        // - https://github.com/gfx-rs/metal-rs/issues/222
504        // - https://news.ycombinator.com/item?id=29311736
505        // - https://stackoverflow.com/a/23765612
506        //
507        // SAFETY:
508        // Based on https://doc.rust-lang.org/stable/reference/inline-assembly.html#rules-for-inline-assembly
509        //
510        // We don't care about the value of the register (so it's okay to be
511        // undefined), and its value is preserved.
512        //
513        // nomem: No reads or writes to memory are performed (this `mov`
514        //   operates entirely on registers).
515        // preserves_flags: `mov` doesn't modify any flags.
516        // nostack: We don't touch the stack.
517
518        // Only worth doing on the Apple runtime.
519        // Not supported on TARGET_OS_WIN32.
520        #[cfg(target_vendor = "apple")]
521        {
522            // Supported since macOS 10.7.
523            #[cfg(target_arch = "x86_64")]
524            {
525                // x86_64 looks at the next call instruction.
526                //
527                // This is expected to be a PLT entry - if the user specifies
528                // `-Zplt=no`, a GOT entry will be created instead, and this
529                // will not work.
530            }
531
532            // Supported since macOS 10.8.
533            #[cfg(target_arch = "arm")]
534            unsafe {
535                core::arch::asm!("mov r7, r7", options(nomem, preserves_flags, nostack))
536            };
537
538            // Supported since macOS 10.10.
539            //
540            // On macOS 13.0 / iOS 16.0 / tvOS 16.0 / watchOS 9.0, the runtime
541            // instead checks the return pointer address, so we no longer need
542            // to emit these extra instructions, see this video from WWDC22:
543            // https://developer.apple.com/videos/play/wwdc2022/110363/
544            #[cfg(all(target_arch = "aarch64", not(feature = "unstable-apple-new")))]
545            unsafe {
546                // Same as `mov x29, x29`.
547                core::arch::asm!("mov fp, fp", options(nomem, preserves_flags, nostack))
548            };
549
550            // Supported since macOS 10.12.
551            #[cfg(target_arch = "x86")]
552            unsafe {
553                core::arch::asm!("mov ebp, ebp", options(nomem, preserves_flags, nostack))
554            };
555        }
556
557        // SAFETY: Same as `Retained::retain`, this is just an optimization.
558        let res: *mut T = unsafe { ffi::objc_retainAutoreleasedReturnValue(ptr.cast()) }.cast();
559
560        // Ideally, we'd be able to specify that the above call should never
561        // be tail-call optimized (become a `jmp` instruction instead of a
562        // `call`); Rust doesn't really have a way of doing this currently, so
563        // we emit a `nop` to make such tail-call optimizations less likely to
564        // occur.
565        //
566        // This is brittle! We should find a better solution!
567        #[cfg(all(target_vendor = "apple", target_arch = "x86_64"))]
568        {
569            // SAFETY: Similar to above.
570            unsafe { core::arch::asm!("nop", options(nomem, preserves_flags, nostack)) };
571            // TODO: Possibly more efficient alternative? Also consider PLT.
572            // #![feature(asm_sym)]
573            // core::arch::asm!(
574            //     "mov rdi, rax",
575            //     "call {}",
576            //     sym objc2::ffi::objc_retainAutoreleasedReturnValue,
577            //     inout("rax") obj,
578            //     clobber_abi("C-unwind"),
579            // );
580        }
581
582        debug_assert_eq!(
583            res, ptr,
584            "objc_retainAutoreleasedReturnValue did not return the same pointer"
585        );
586
587        // SAFETY: Same as `Retained::retain`.
588        unsafe { Self::from_raw(res) }
589    }

591    /// Autoreleases the [`Retained`], returning a pointer.
592    ///
593    /// The object is not immediately released, but will be when the innermost
594    /// / current autorelease pool is drained.
595    ///
596    /// This is useful when defining your own classes and you have some error
597    /// parameter passed as `Option<&mut *mut NSError>`, and you want to
598    /// create and autorelease an error before returning.
599    ///
600    /// This is an associated method, and must be called as
601    /// `Retained::autorelease_ptr(obj)`.
602    ///
603    /// # Safety
604    ///
605    /// This method is safe to call, but the returned pointer is only
606    /// guaranteed to be valid until the innermost autorelease pool is
607    /// drained.
608    #[doc(alias = "objc_autorelease")]
609    #[must_use = "if you don't intend to use the object any more, drop it as usual"]
610    #[inline]
611    pub fn autorelease_ptr(this: Self) -> *mut T {
612        let ptr = ManuallyDrop::new(this).ptr.as_ptr();
613        // SAFETY:
614        // - The `ptr` is guaranteed to be valid and have at least one
615        //   retain count.
616        // - Because of the ManuallyDrop, we don't call the Drop
617        //   implementation, so the object won't also be released there.
618        let res: *mut T = unsafe { ffi::objc_autorelease(ptr.cast()) }.cast();
619        debug_assert_eq!(res, ptr, "objc_autorelease did not return the same pointer");
620        res
621    }
622
623    /// Autoreleases the [`Retained`], returning a reference bound to the pool.
624    ///
625    /// The object is not immediately released, but will be when the innermost
626    /// / current autorelease pool (given as a parameter) is drained.
627    ///
628    /// This is an associated method, and must be called as
629    /// `Retained::autorelease(obj, pool)`.
630    ///
631    /// # Safety
632    ///
633    /// The given pool must represent the innermost pool, to ensure that the
634    /// reference is not moved outside the autorelease pool into which it has
635    /// been put in.
636    #[doc(alias = "objc_autorelease")]
637    #[must_use = "if you don't intend to use the object any more, drop it as usual"]
638    #[inline]
639    #[allow(clippy::needless_lifetimes)]
640    pub unsafe fn autorelease<'p>(this: Self, pool: AutoreleasePool<'p>) -> &'p T {
641        let ptr = Self::autorelease_ptr(this);
642        // SAFETY: The pointer is valid as a reference
643        unsafe { pool.ptr_as_ref(ptr) }
644    }
645
646    #[inline]
647    pub(crate) fn autorelease_return_option(this: Option<Self>) -> *mut T {
648        let ptr: *mut T = this
649            .map(|this| ManuallyDrop::new(this).ptr.as_ptr())
650            .unwrap_or_else(ptr::null_mut);
651
652        // SAFETY: Same as `autorelease_inner`, this is just an optimization.
653        let res: *mut T = unsafe { ffi::objc_autoreleaseReturnValue(ptr.cast()) }.cast();
654        debug_assert_eq!(
655            res, ptr,
656            "objc_autoreleaseReturnValue did not return the same pointer"
657        );
658        res
659    }

661    /// Autoreleases and prepares the [`Retained`] to be returned to Objective-C.
662    ///
663    /// The object is not immediately released, but will be when the innermost
664    /// autorelease pool is drained.
665    ///
666    /// This is useful when [defining your own methods][classbuilder] where
667    /// you will often find yourself in need of returning autoreleased objects
668    /// to properly follow [Cocoa's Memory Management Policy][mmRules].
669    ///
670    /// To that end, you could also use [`Retained::autorelease_ptr`], but
671    /// this is more efficient than a normal `autorelease`, since it makes a
672    /// best effort attempt to hand off ownership of the retain count to a
673    /// subsequent call to `objc_retainAutoreleasedReturnValue` /
674    /// [`Retained::retain_autoreleased`] in the enclosing call frame.
675    ///
676    /// This optimization relies heavily on this function being tail called,
677    /// so make sure you only call this function at the end of your method.
678    ///
679    /// [classbuilder]: crate::runtime::ClassBuilder
680    /// [mmRules]: https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/MemoryMgmt/Articles/mmRules.html
681    ///
682    ///
683    /// # Example
684    ///
685    /// Returning an `Retained` from a custom method (note: the
686    /// [`define_class!`] macro supports doing this for you automatically).
687    ///
688    /// ```
689    /// use objc2::{class, msg_send, sel};
690    /// use objc2::rc::Retained;
691    /// use objc2::runtime::{AnyClass, AnyObject, ClassBuilder, Sel};
692    ///
693    /// let mut builder = ClassBuilder::new(c"ExampleObject", class!(NSObject)).unwrap();
694    ///
695    /// extern "C-unwind" fn get(cls: &AnyClass, _cmd: Sel) -> *mut AnyObject {
696    ///     let obj: Retained<AnyObject> = unsafe { msg_send![cls, new] };
697    ///     Retained::autorelease_return(obj)
698    /// }
699    ///
700    /// unsafe {
701    ///     builder.add_class_method(
702    ///         sel!(get),
703    ///         get as extern "C-unwind" fn(_, _) -> _,
704    ///     );
705    /// }
706    ///
707    /// let cls = builder.register();
708    /// ```
709    ///
710    /// [`define_class!`]: crate::define_class
711    #[doc(alias = "objc_autoreleaseReturnValue")]
712    #[must_use = "if you don't intend to use the object any more, drop it as usual"]
713    #[inline]
714    pub fn autorelease_return(this: Self) -> *mut T {
715        Self::autorelease_return_option(Some(this))
716    }
717}

724    //
725    // NOTE: This is _not_ an associated method, since we want it to be easy

752/// `#[may_dangle]` (see [this][dropck_eyepatch]) doesn't apply here since we
753/// don't run `T`'s destructor (rather, we want to discourage having `T`s with
754/// a destructor); and even if we did run the destructor, it would not be safe
755/// to add since we cannot verify that a `dealloc` method doesn't access
756/// borrowed data.
757///
758/// [dropck_eyepatch]: https://doc.rust-lang.org/nightly/nomicon/dropck.html#an-escape-hatch
759impl<T: ?Sized> Drop for Retained<T> {
760    /// Releases the retained object.
761    ///
762    /// The contained object's destructor (`Drop` impl, if it has one) is
763    /// never run - override the `dealloc` method instead (which
764    /// `define_class!` does for you).
765    #[doc(alias = "objc_release")]
766    #[doc(alias = "release")]
767    #[inline]
768    fn drop(&mut self) {
769        // We could technically run the destructor for `T` when it is mutable,
770        // but that would be confusing and inconsistent since we cannot really
771        // guarantee that it is run if the `Retained<T>` is passed to Objective-C.
772
773        // SAFETY: The `ptr` is guaranteed to be valid and have at least one
774        // retain count.
775        unsafe { objc_release_fast(self.ptr.as_ptr().cast()) };
776    }
777}

799// `Retained`, as it conflicts with the generic `impl From<T> for T`.
800//
801// impl<T: Upcast, U> From<Retained<U>> for Retained<T> {

814    }

837/// `Retained<T>` is `Send` if `T` is `Send + Sync`.
838//
839// SAFETY:
840// - `T: Send` is required because otherwise you could move the object to
841//   another thread and let `dealloc` get called there.
842// - `T: Sync` is required because otherwise you could clone `&Retained<T>`,
843//   send it to another thread, and drop the clone last, making `dealloc` get
844//   called on the other thread.
845//
846// This is the same reasoning as for `Arc`.
847// https://doc.rust-lang.org/nomicon/arc-mutex/arc-base.html#send-and-sync
848unsafe impl<T: ?Sized + Sync + Send> Send for Retained<T> {}
849
850/// `Retained<T>` is `Sync` if `T` is `Send + Sync`.
851//
852// SAFETY:
853// - `T: Sync` is required because `&Retained<T>` give access to `&T`.
854// -`T: Send` is required because otherwise you could clone `&Retained<T>`
855//   from another thread, and drop the clone last, making `dealloc` get called
856//   on the other thread.
857//
858// This is the same reasoning as for `Arc`.
859// https://doc.rust-lang.org/nomicon/arc-mutex/arc-base.html#send-and-sync
860unsafe impl<T: ?Sized + Sync + Send> Sync for Retained<T> {}
861
862// This is valid without `T: Unpin` because we don't implement any projection.
863//
864// See https://doc.rust-lang.org/1.54.0/src/alloc/boxed.rs.html#1652-1675
865// and the `Arc` implementation.
866impl<T: ?Sized> Unpin for Retained<T> {}
