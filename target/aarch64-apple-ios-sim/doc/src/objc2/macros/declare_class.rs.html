1/// Declare a new class.
2///
3/// This is useful in many cases since Objective-C frameworks tend to favour a
4/// design pattern using "delegates", where to hook into a piece of
5/// functionality in a class, you implement that class' delegate protocol in
6/// a custom class.
7///
8/// This macro is the declarative way of creating classes, in contrast with
9/// the [`declare`] module which mostly contain ways of declaring classes in
10/// an imperative fashion. It is highly recommended that you use this macro
11/// though, since it contains a lot of extra debug assertions and niceties
12/// that help ensure the soundness of your code.
13///
14/// [`declare`]: crate::declare
15///
16///
17/// # Specification
18///
19/// This macro consists of the following parts (the first three are required):
20/// - The type declaration.
21/// - The [`ClassType`] implementation.
22/// - The [`DeclaredClass`] implementation.
23/// - Any number of inherent implementations.
24/// - Any number of protocol implementations.
25///
26/// With the syntax generally resembling a combination of that in
27/// [`extern_class!`] and [`extern_methods!`].
28///
29/// [`ClassType`]: crate::ClassType
30/// [`DeclaredClass`]: crate::DeclaredClass
31/// [`extern_class!`]: crate::extern_class
32/// [`extern_methods!`]: crate::extern_methods
33///
34///
35/// ## Type declaration
36///
37/// The type declaration works a lot like in [`extern_class!`], an opaque
38/// struct is created and a lot of traits is implemented for that struct.
39///
40/// You are allowed to add most common attributes to the declaration,
41/// including `#[cfg(...)]` and doc comments. ABI-modifying attributes like
42/// `#[repr(...)]` are not allowed.
43///
44/// `#[derive(...)]` attributes are allowed, but heavily discouraged, as they
45/// are likely to not work as you'd expect them to. This is being worked on in
46/// [#267].
47///
48/// If the type implements [`Drop`], the macro will generate a `dealloc`
49/// method for you, which will call `drop` automatically.
50///
51/// [#267]: https://github.com/madsmtm/objc2/issues/267
52///
53///
54/// ## `ClassType` implementation
55///
56/// This also resembles the syntax in [`extern_class!`], except that
57/// [`ClassType::NAME`] must be specified, and it must be unique across the
58/// entire application.
59///
60/// If you're developing a library, good practice here would be to include
61/// your crate name in the prefix (something like `"MyLibrary_MyClass"`).
62///
63/// The class is guaranteed to have been created and registered with the
64/// Objective-C runtime after the [`ClassType::class`] function has been
65/// called.
66///
67/// [`ClassType::NAME`]: crate::ClassType::NAME
68/// [`ClassType::class`]: crate::ClassType::class
69///
70///
71/// ## `DeclaredClass` implementation
72///
73/// The syntax here is as if you were implementing the trait yourself.
74///
75/// You may optionally specify the associated type [`Ivars`]; this is the
76/// intended way to specify the data your class stores. If you don't specify
77/// any ivars, the macro will default to [`()`][unit].
78///
79/// Beware that if you want to use the class' inherited initializers (such as
80/// `init`), you must override the subclass' designated initializers, and
81/// initialize your ivars properly in there.
82///
83/// [`Ivars`]: crate::DeclaredClass::Ivars
84///
85///
86/// ## Inherent method definitions
87///
88/// Within the `impl` block you can define two types of functions;
89/// ["associated functions"] and ["methods"]. These are then mapped to the
90/// Objective-C equivalents "class methods" and "instance methods". In
91/// particular, if you use `self` or the special name `this` (or `_this`),
92/// your method will be registered as an instance method, and if you don't it
93/// will be registered as a class method.
94///
95/// On instance methods, you can freely choose between different types of
96/// receivers, e.g. `&self`, `this: *const Self`, `&mut self`, and so on. Note
97/// though that using `&mut self` requires the class' mutability to be
98/// [`IsAllowedMutable`].
99/// If you need mutation of your class' instance variables, consider using
100/// [`Cell`] or similar instead.
101///
102/// The desired selector can be specified using the `#[method(my:selector:)]`
103/// or `#[method_id(my:selector:)]` attribute, similar to the
104/// [`extern_methods!`] macro.
105///
106/// If the `#[method_id(...)]` attribute is used, the return type must be
107/// `Option<Retained<T>>` or `Retained<T>`. Additionally, if the selector is
108/// in the "init"-family, the `self`/`this` parameter must be
109/// `Allocated<Self>`.
110///
111/// Putting other attributes on the method such as `cfg`, `allow`, `doc`,
112/// `deprecated` and so on is supported. However, note that `cfg_attr` may not
113/// work correctly, due to implementation difficulty - if you have a concrete
114/// use-case, please [open an issue], then we can discuss it.
115///
116/// A transformation step is performed on the functions (to make them have the
117/// correct ABI) and hence they shouldn't really be called manually. (You
118/// can't mark them as `pub` for the same reason). Instead, use the
119/// [`extern_methods!`] macro to create a Rust interface to the methods.
120///
121/// If the parameter or return type is [`bool`], a conversion is performed to
122/// make it behave similarly to the Objective-C `BOOL`. Use [`runtime::Bool`]
123/// if you want to control this manually.
124///
125/// Note that `&mut Retained<_>` and other such out parameters are not yet
126/// supported, and may generate a panic at runtime.
127///
128/// ["associated functions"]: https://doc.rust-lang.org/reference/items/associated-items.html#methods
129/// ["methods"]: https://doc.rust-lang.org/reference/items/associated-items.html#methods
130/// [`IsAllowedMutable`]: crate::mutability::IsAllowedMutable
131/// [`Cell`]: core::cell::Cell
132/// [open an issue]: https://github.com/madsmtm/objc2/issues/new
133/// [`msg_send!`]: crate::msg_send
134/// [`runtime::Bool`]: crate::runtime::Bool
135///
136///
137/// ## Protocol implementations
138///
139/// You can specify protocols that the class should implement, along with any
140/// required/optional methods for said protocols.
141///
142/// The protocol must have been previously defined with [`extern_protocol!`].
143///
144/// The methods work exactly as normal, they're only put "under" the protocol
145/// definition to make things easier to read.
146///
147/// Putting attributes on the `impl` item such as `cfg`, `allow`, `doc`,
148/// `deprecated` and so on is supported.
149///
150/// [`extern_protocol!`]: crate::extern_protocol
151///
152///
153/// # Panics
154///
155/// The implemented `ClassType::class` method may panic in a few cases, such
156/// as if:
157/// - A class with the specified name already exists.
158/// - Debug assertions are enabled, and an overriden method's signature is not
159///   equal to the one on the superclass.
160/// - Debug assertions are enabled, and the protocol's required methods are not
161///   implemented.
162///
163/// And possibly more similar cases in the future.
164///
165///
166/// # Safety
167///
168/// Using this macro requires writing a few `unsafe` markers:
169///
170/// `unsafe impl ClassType for T` has the following safety requirements:
171/// - Any invariants that the superclass [`ClassType::Super`] may have must be
172///   upheld.
173/// - [`ClassType::Mutability`] must be correct.
174/// - If your type implements `Drop`, the implementation must abide by the
175///   following rules:
176///   - It must not call any overridden methods.
177///   - It must not `retain` the object past the lifetime of the drop.
178///   - It must not `retain` in the same scope that `&mut self` is active.
179///   - TODO: And probably a few more. [Open an issue] if you would like
180///     guidance on whether your implementation is correct.
181///
182/// `unsafe impl T { ... }` asserts that the types match those that are
183/// expected when the method is invoked from Objective-C. Note that unlike
184/// with [`extern_methods!`], there are no safe-guards here; you can write
185/// `i8`, but if Objective-C thinks it's an `u32`, it will cause UB when
186/// called!
187///
188/// `unsafe impl P for T { ... }` requires that all required methods of the
189/// specified protocol is implemented, and that any extra requirements
190/// (implicit or explicit) that the protocol has are upheld. The methods in
191/// this definition has the same safety requirements as above.
192///
193/// [`ClassType::Super`]: crate::ClassType::Super
194/// [`ClassType::Mutability`]: crate::ClassType::Mutability
195/// [Open an issue]: https://github.com/madsmtm/objc2/issues/new
196///
197///
198/// # Examples
199///
200/// Declare a class `MyCustomObject` that inherits `NSObject`, has a few
201/// instance variables and methods, and implements the `NSCopying` protocol.
202///
203/// ```
204/// use std::os::raw::c_int;
205///
206/// # use objc2::runtime::{NSObject, NSObjectProtocol, NSZone};
207/// # #[cfg(available_in_foundation)]
208/// use objc2_foundation::{NSCopying, NSObject, NSObjectProtocol, NSZone};
209/// use objc2::rc::{Allocated, Retained};
210/// use objc2::{
211///     declare_class, extern_protocol, msg_send, msg_send_id, mutability, ClassType,
212///     DeclaredClass, ProtocolType,
213/// };
214///
215/// #[derive(Clone)]
216/// struct Ivars {
217///     foo: u8,
218///     bar: c_int,
219///     object: Retained<NSObject>,
220/// }
221///
222/// declare_class!(
223///     struct MyCustomObject;
224///
225///     // SAFETY:
226///     // - The superclass NSObject does not have any subclassing requirements.
227///     // - Interior mutability is a safe default.
228///     // - `MyCustomObject` does not implement `Drop`.
229///     unsafe impl ClassType for MyCustomObject {
230///         type Super = NSObject;
231///         type Mutability = mutability::InteriorMutable;
232///         const NAME: &'static str = "MyCustomObject";
233///     }
234///
235///     impl DeclaredClass for MyCustomObject {
236///         type Ivars = Ivars;
237///     }
238///
239///     unsafe impl MyCustomObject {
240///         #[method_id(initWithFoo:)]
241///         fn init_with(this: Allocated<Self>, foo: u8) -> Option<Retained<Self>> {
242///             let this = this.set_ivars(Ivars {
243///                 foo,
244///                 bar: 42,
245///                 object: NSObject::new(),
246///             });
247///             unsafe { msg_send_id![super(this), init] }
248///         }
249///
250///         #[method(foo)]
251///         fn __get_foo(&self) -> u8 {
252///             self.ivars().foo
253///         }
254///
255///         #[method_id(object)]
256///         fn __get_object(&self) -> Retained<NSObject> {
257///             self.ivars().object.clone()
258///         }
259///
260///         #[method(myClassMethod)]
261///         fn __my_class_method() -> bool {
262///             true
263///         }
264/// #
265/// #       #[method_id(copyWithZone:)]
266/// #       fn copyWithZone(&self, _zone: *const NSZone) -> Retained<Self> {
267/// #           let new = Self::alloc().set_ivars(self.ivars().clone());
268/// #           unsafe { msg_send_id![super(new), init] }
269/// #       }
270///     }
271///
272///     unsafe impl NSObjectProtocol for MyCustomObject {}
273///
274///     # #[cfg(available_in_foundation)]
275///     unsafe impl NSCopying for MyCustomObject {
276///         #[method_id(copyWithZone:)]
277///         fn copyWithZone(&self, _zone: *const NSZone) -> Retained<Self> {
278///             let new = Self::alloc().set_ivars(self.ivars().clone());
279///             unsafe { msg_send_id![super(new), init] }
280///         }
281///
282///         // If we have tried to add other methods here, or had forgotten
283///         // to implement the method, we would have gotten an error.
284///     }
285/// );
286///
287/// impl MyCustomObject {
288///     pub fn new(foo: u8) -> Retained<Self> {
289///         unsafe { msg_send_id![Self::alloc(), initWithFoo: foo] }
290///     }
291///
292///     pub fn get_foo(&self) -> u8 {
293///         unsafe { msg_send![self, foo] }
294///     }
295///
296///     pub fn get_object(&self) -> Retained<NSObject> {
297///         unsafe { msg_send_id![self, object] }
298///     }
299///
300///     pub fn my_class_method() -> bool {
301///         unsafe { msg_send![Self::class(), myClassMethod] }
302///     }
303/// }
304///
305/// fn main() {
306///     let obj = MyCustomObject::new(3);
307///     assert_eq!(obj.ivars().foo, 3);
308///     assert_eq!(obj.ivars().bar, 42);
309///     assert!(obj.ivars().object.is_kind_of::<NSObject>());
310///
311/// #   let obj: Retained<MyCustomObject> = unsafe { msg_send_id![&obj, copy] };
312/// #   #[cfg(available_in_foundation)]
313///     let obj = obj.copy();
314///
315///     assert_eq!(obj.get_foo(), 3);
316///     assert!(obj.get_object().is_kind_of::<NSObject>());
317///
318///     assert!(MyCustomObject::my_class_method());
319/// }
320/// ```
321///
322/// Approximately equivalent to the following ARC-enabled Objective-C code.
323///
324/// ```text
325/// #import <Foundation/Foundation.h>
326///
327/// @interface MyCustomObject: NSObject <NSCopying>
328/// - (instancetype)initWithFoo:(uint8_t)foo;
329/// - (uint8_t)foo;
330/// - (NSObject*)object;
331/// + (BOOL)myClassMethod;
332/// @end
333///
334///
335/// @implementation MyCustomObject {
336///     // Instance variables
337///     uint8_t foo;
338///     int bar;
339///     NSObject* _Nonnull object;
340/// }
341///
342/// - (instancetype)initWithFoo:(uint8_t)foo_arg {
343///     self = [super init];
344///     if (self) {
345///         self->foo = foo_arg;
346///         self->bar = 42;
347///         self->object = [NSObject new];
348///     }
349///     return self;
350/// }
351///
352/// - (uint8_t)foo {
353///     return self->foo;
354/// }
355///
356/// - (NSObject*)object {
357///     return self->object;
358/// }
359///
360/// + (BOOL)myClassMethod {
361///     return YES;
362/// }
363///
364/// // NSCopying
365///
366/// - (id)copyWithZone:(NSZone *)_zone {
367///     MyCustomObject* new = [[MyCustomObject alloc] initWithFoo: self->foo];
368///     new->bar = self->bar;
369///     new->obj = self->obj;
370///     return new;
371/// }
372///
373/// @end
374/// ```

377#[macro_export]
378macro_rules! declare_class {
379    {
380        $(#[$m:meta])*
381        $v:vis struct $name:ident;
382
383        unsafe impl ClassType for $for_class:ty {
384            $(#[inherits($($inheritance_rest:ty),+)])?
385            type Super = $superclass:ty;
386
387            type Mutability = $mutability:ty;
388
389            const NAME: &'static str = $name_const:expr;
390        }
391
392        impl DeclaredClass for $for_declared:ty {
393            $(type Ivars = $ivars:ty;)?
394        }
395
396        $($impls:tt)*
397    } => {
398        $(#[$m])*
399        #[repr(C)]
400        $v struct $name {
401            // Superclasses are deallocated by calling `[super dealloc]`.
402            __superclass: $crate::__macro_helpers::ManuallyDrop<$superclass>,
403            // Include ivars for proper auto traits.
404            __ivars: $crate::__macro_helpers::PhantomData<<Self as $crate::DeclaredClass>::Ivars>,
405        }
406
407        $crate::__extern_class_impl_traits! {
408            // SAFETY: Upheld by caller
409            unsafe impl () for $for_class {
410                INHERITS = [$superclass, $($($inheritance_rest,)+)? $crate::runtime::AnyObject];
411
412                fn as_super(&self) {
413                    &*self.__superclass
414                }
415
416                fn as_super_mut(&mut self) {
417                    &mut *self.__superclass
418                }
419            }
420        }
421
422        // Anonymous block to hide the shared statics
423        const _: () = {
424            static mut __OBJC2_CLASS: $crate::__macro_helpers::MaybeUninit<&'static $crate::runtime::AnyClass> = $crate::__macro_helpers::MaybeUninit::uninit();
425            static mut __OBJC2_IVAR_OFFSET: $crate::__macro_helpers::MaybeUninit<$crate::__macro_helpers::isize> = $crate::__macro_helpers::MaybeUninit::uninit();
426            static mut __OBJC2_DROP_FLAG_OFFSET: $crate::__macro_helpers::MaybeUninit<$crate::__macro_helpers::isize> = $crate::__macro_helpers::MaybeUninit::uninit();
427
428            // Creation
429            unsafe impl ClassType for $for_class {
430                type Super = $superclass;
431                type Mutability = $mutability;
432                const NAME: &'static $crate::__macro_helpers::str = $name_const;
433
434                fn class() -> &'static $crate::runtime::AnyClass {
435                    $crate::__macro_helpers::assert_mutability_matches_superclass_mutability::<Self>();
436
437                    // TODO: Use `std::sync::OnceLock`
438                    static REGISTER_CLASS: $crate::__macro_helpers::Once = $crate::__macro_helpers::Once::new();
439
440                    REGISTER_CLASS.call_once(|| {
441                        let mut __objc2_builder = $crate::__macro_helpers::ClassBuilderHelper::<Self>::new();
442
443                        // Implement protocols and methods
444                        $crate::__declare_class_register_impls! {
445                            (__objc2_builder)
446                            $($impls)*
447                        }
448
449                        let (__objc2_cls, __objc2_ivar_offset, __objc2_drop_flag_offset) = __objc2_builder.register();
450
451                        // SAFETY: Modification is ensured by `Once` to happen
452                        // before any access to the variables.
453                        unsafe {
454                            __OBJC2_CLASS.write(__objc2_cls);
455                            if <Self as $crate::__macro_helpers::DeclaredIvarsHelper>::HAS_IVARS {
456                                __OBJC2_IVAR_OFFSET.write(__objc2_ivar_offset);
457                            }
458                            if <Self as $crate::__macro_helpers::DeclaredIvarsHelper>::HAS_DROP_FLAG {
459                                __OBJC2_DROP_FLAG_OFFSET.write(__objc2_drop_flag_offset);
460                            }
461                        }
462                    });
463
464                    // SAFETY: We just registered the class, so is now available
465                    unsafe { __OBJC2_CLASS.assume_init() }
466                }
467
468                #[inline]
469                fn as_super(&self) -> &Self::Super {
470                    &*self.__superclass
471                }
472
473                #[inline]
474                fn as_super_mut(&mut self) -> &mut Self::Super {
475                    &mut *self.__superclass
476                }
477            }
478
479            impl DeclaredClass for $for_declared {
480                type Ivars = $crate::__select_ivars!($($ivars)?);
481
482                #[inline]
483                fn __ivars_offset() -> $crate::__macro_helpers::isize {
484                    // Only access ivar offset if we have an ivar.
485                    //
486                    // This makes the offset not be included in the final
487                    // executable if it's not needed.
488                    if <Self as $crate::__macro_helpers::DeclaredIvarsHelper>::HAS_IVARS {
489                        // SAFETY: Accessing the offset is guaranteed to only be
490                        // done after the class has been initialized.
491                        unsafe { __OBJC2_IVAR_OFFSET.assume_init() }
492                    } else {
493                        // Fall back to an offset of zero.
494                        //
495                        // This is fine, since any reads here will only be via. zero-sized
496                        // ivars, where the actual pointer doesn't matter.
497                        0
498                    }
499                }
500
501                #[inline]
502                fn __drop_flag_offset() -> $crate::__macro_helpers::isize {
503                    if <Self as $crate::__macro_helpers::DeclaredIvarsHelper>::HAS_DROP_FLAG {
504                        // SAFETY: Same as above.
505                        unsafe { __OBJC2_DROP_FLAG_OFFSET.assume_init() }
506                    } else {
507                        // Fall back to an offset of zero.
508                        //
509                        // This is fine, since the drop flag is never actually used in the
510                        // cases where it was not added.
511                        0
512                    }
513                }
514
515                // SAFETY: The offsets are implemented correctly
516                const __UNSAFE_OFFSETS_CORRECT: () = ();
517            }
518        };
519
520        // Methods
521        $crate::__declare_class_output_impls! {
522            $($impls)*
523        }
524    };
525}

660#[doc(hidden)]
661#[macro_export]
662macro_rules! __declare_class_register_impls {
663    // Base-case
664    (
665        ($builder:ident)
666    ) => {};
667
668    // With protocol
669    (
670        ($builder:ident)
671
672        $(#[$($m:tt)*])*
673        unsafe impl $protocol:ident for $for:ty {
674            $($methods:tt)*
675        }
676
677        $($rest:tt)*
678    ) => {
679        $crate::__extract_and_apply_cfg_attributes! {
680            ($(#[$($m)*])*)
681
682            // Implement protocol
683            #[allow(unused_mut)]
684            let mut __objc2_protocol_builder = $builder.add_protocol_methods::<dyn $protocol>();
685
686            // In case the user's function is marked `deprecated`
687            #[allow(deprecated)]
688            // In case the user did not specify any methods
689            #[allow(unused_unsafe)]
690            // SAFETY: Upheld by caller
691            unsafe {
692                $crate::__declare_class_register_methods! {
693                    (__objc2_protocol_builder)
694
695                    $($methods)*
696                }
697            }
698
699            // Finished declaring protocol; get error message if any
700            __objc2_protocol_builder.finish();
701        }
702
703        $crate::__declare_class_register_impls! {
