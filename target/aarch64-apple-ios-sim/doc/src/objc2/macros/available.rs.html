1/// Check if APIs from the given operating system versions are available.
2///
3/// Apple adds new APIs with new OS releases, and as a developer, you often
4/// want to use those to give your users the best behaviour, while still
5/// supporting older OS versions that don't have those APIs (instead of
6/// crashing e.g. because of an undefined selector).
7///
8/// This macro allows you to conditionally execute code depending on if the
9/// current OS version is higher than or equal to the version given in the
10/// macro.
11///
12/// If no version is specified for a certain OS, the API will be assumed to be
13/// unavailable there. This default can be changed by adding a trailing `..`
14/// to the macro invocation.
15///
16/// This is very similar to `@available` in Objective-C and `#available` in
17/// Swift, see [Apple's documentation][apple-doc]. Another great introduction
18/// to availability can be found in [here][epir-availability].
19///
20/// [apple-doc]: https://developer.apple.com/documentation/xcode/running-code-on-a-specific-version#Require-a-minimum-operating-system-version-for-a-feature
21/// [epir-availability]: https://epir.at/2019/10/30/api-availability-and-target-conditionals/
22///
23///
24/// # Operating systems
25///
26/// The operating system names this macro accepts, the standard environment
27/// variables that you use to raise the deployment target (the minimum
28/// supported OS version) and the current default versions are all summarized
29/// in the table below.
30///
31/// | OS Value   | Name                    | Environment Variable         | Default |
32/// | ---------- | ----------------------- | ---------------------------- | ------- |
33/// | `ios`      | iOS/iPadOS/Mac Catalyst | `IPHONEOS_DEPLOYMENT_TARGET` | 10.0    |
34/// | `macos`    | macOS                   | `MACOSX_DEPLOYMENT_TARGET`   | 10.12   |
35/// | `tvos`     | tvOS                    | `TVOS_DEPLOYMENT_TARGET`     | 10.0    |
36/// | `visionos` | visionOS                | `XROS_DEPLOYMENT_TARGET`     | 1.0     |
37/// | `watchos`  | watchOS                 | `WATCHOS_DEPLOYMENT_TARGET`  | 5.0     |
38///
39/// The default version is the same as that of `rustc` itself.
40///
41///
42/// # Optimizations
43///
44/// This macro will statically be set to `true` when the deployment target is
45/// high enough.
46///
47/// If a runtime check is deemed necessary, the version lookup will be cached.
48///
49///
50/// # Alternatives
51///
52/// Instead of checking the version at runtime, you could do one of the
53/// following instead:
54///
55/// 1. Check statically that you're compiling for a version where the API is
56///    available, e.g. by checking the `*_DEPLOYMENT_TARGET` variables in a
57///    build script or at `const` time.
58///
59/// 2. Check at runtime that a class, method or symbol is available, using
60///    e.g. [`AnyClass::get`], [`respondsToSelector`] or [weak linking].
61///
62/// [`AnyClass::get`]: crate::runtime::AnyClass::get
63/// [`respondsToSelector`]: crate::runtime::NSObjectProtocol::respondsToSelector
64/// [weak linking]: https://github.com/rust-lang/rust/issues/29603
65///
66///
67/// # Examples
68///
69/// Use the [`effectiveAppearance`] API that was added in macOS 10.14.
70///
71/// ```
72/// # #[cfg(available_in_frameworks)]
73/// use objc2_app_kit::{NSApplication, NSAppearance, NSAppearanceNameAqua};
74/// use objc2::available;
75///
76/// let appearance = if available!(macos = 10.14) {
77///     // Dark mode and `effectiveAppearance` was added in macOS 10.14.
78///     # #[cfg(available_in_frameworks)]
79///     NSApplication::sharedApplication(mtm).effectiveAppearance()
80/// } else {
81///     // Fall back to `NSAppearanceNameAqua` on macOS 10.13 and below.
82///     # #[cfg(available_in_frameworks)]
83///     NSAppearance::appearanceNamed(NSAppearanceNameAqua).unwrap()
84/// };
85/// ```
86///
87/// Use an API added in Xcode 16.0 SDKs.
88///
89/// We use `..` here in case Apple adds a new operating system in the future,
90/// then we probably also want the branch to be taken there.
91///
92/// ```
93/// use objc2::available;
94///
95/// if available!(ios = 18.0, macos = 15.0, tvos = 18.0, visionos = 2.0, watchos = 11.0, ..) {
96///     // Use some recent API here.
97/// }
98/// ```
99///
100/// Set the [`wantsExtendedDynamicRangeContent`] property, which is available
101/// since iOS 16.0, macOS 10.11 and visionOS 1.0, but is not available on tvOS
102/// and watchOS.
103///
104/// ```
105/// use objc2::available;
106///
107/// if available!(ios = 16.0, macos = 10.11, visionos = 1.0) {
108///     # #[cfg(available_in_frameworks)]
109///     layer.setWantsExtendedDynamicRangeContent(true);
110/// }
111/// ```
112///
113/// Work around problems in a specific range of versions (an example of this
114/// in the real world can be seen in [#662]).
115///
116/// ```
117/// use objc2::available;
118///
119/// if available!(macos = 15.0) && !available!(macos = 15.1) {
120///     // Do something on macOS 15.0 and 15.0.1.
121/// } else {
122///     // Do something else on all other versions.
123/// }
124/// ```
125///
126/// [`effectiveAppearance`]: https://developer.apple.com/documentation/appkit/nsapplication/2967171-effectiveappearance?language=objc
127/// [`wantsExtendedDynamicRangeContent`]: https://developer.apple.com/documentation/quartzcore/cametallayer/1478161-wantsextendeddynamicrangecontent
128/// [#662]: https://github.com/madsmtm/objc2/issues/662
129#[doc(alias = "@available")] // Objective-C
130#[doc(alias = "#available")] // Swift
131#[macro_export]
132macro_rules! available {
133    (
134        // Returns `false` on unspecified platforms.
135        $(
136            $os:ident $(= $major:literal $(. $minor:literal $(. $patch:literal)?)?)?
137        ),* $(,)?
138    ) => {
139        $crate::__macro_helpers::is_available({
140            // TODO: Use inline const once in MSRV
141            #[allow(clippy::needless_update)]
142            const VERSION: $crate::__macro_helpers::AvailableVersion = $crate::__macro_helpers::AvailableVersion {
143                $(
144                    // Doesn't actually parse versions this way, but is
145                    // helpful to write it like this for documentation.
146                    //
147                    // We use optionality for the version here, to allow
148                    // rust-analyzer to work with partially filled macros.
149                    $os: $($crate::__available_version!($major $(. $minor $(. $patch)?)?))?,
150                )*
151                // A version this high will never be lower than the deployment
152                // target, and hence will always return `false` from
153                // `is_available`.
154                .. $crate::__macro_helpers::AvailableVersion::MAX
155            };
156            VERSION
157        })
158    };
159    (
160        // Returns `true` on unspecified platforms because of the trailing `..`.
161        $(
162            $os:ident $(= $major:literal $(. $minor:literal $(. $patch:literal)?)?)?,
163        )*
164        ..
165    ) => {
166        $crate::__macro_helpers::is_available({
167            #[allow(clippy::needless_update)]
168            const VERSION: $crate::__macro_helpers::AvailableVersion = $crate::__macro_helpers::AvailableVersion {
169                $(
170                    $os: $($crate::__available_version!($major $(. $minor $(. $patch)?)?))?,
171                )*
172                // A version of 0.0.0 will always be lower than the deployment
173                // target, and hence will always return `true` from
174                // `is_available`.
175                //
176                // We do this when `..` is specified.
177                .. $crate::__macro_helpers::AvailableVersion::MIN
178            };
179            VERSION
180        })
181    };
182}

184/// Both `tt` and `literal` matches either `$major` as an integer, or
185/// `$major.$minor` as a float.
186///
187/// As such, we cannot just take `$major:tt . $minor:tt . $patch:tt` and
188/// convert that to `OSVersion` directly, we must convert it to a string
189/// first, and then parse that.
190///
191/// We also _have_ to do string parsing, floating point parsing wouldn't be
192/// enough (because e.g. `10.10` would result in the float `10.1` and parse
193/// wrongly).
194///
195/// Note that we intentionally `stringify!` before passing to `concat!`, as
196/// that seems to properly preserve all zeros in the literal.
197#[doc(hidden)]
198#[macro_export]
199macro_rules! __available_version {
200    // Just in case rustc's parsing changes in the future, let's handle this
201    // generically, instead of trying to split each part into separate `tt`.
202    ($($version_part_or_period:tt)*) => {
203        $crate::__macro_helpers::OSVersion::from_str($crate::__macro_helpers::concat!($(
204            $crate::__macro_helpers::stringify!($version_part_or_period),
205        )*))
206    };
207}
