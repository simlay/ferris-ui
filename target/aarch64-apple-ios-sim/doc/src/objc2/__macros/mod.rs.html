1//! Declarative macros in `objc2`.
2//!
3//! These are... quite complex, especially the larger ones. Some of this
4//! complexity is inherent, the function rewriting that we have do in
5//! `declare_class!` for example _is_ weird and special, but a lot of it also
6//! just comes from not using procedural macros (which is a requirement for
7//! keeping compile-times down, and to reduce the number of things we run at
8//! build-time).
9//!
10//! Recommended resources (we use all of the patterns described therein):
11//! - [The Little Book of Rust Macros](https://lukaswirth.dev/tlborm/introduction.html)
12//! - [Anything you can do, I can do worse with macro_rules!](https://www.youtube.com/watch?v=7uSM60jlkBE)
13//! - [Aurorans Solis' Blog](https://auroranssolis.github.io/)
