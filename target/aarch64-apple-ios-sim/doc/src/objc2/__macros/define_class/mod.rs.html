9/// Create a new Objective-C class.
10///
11/// This is useful in many cases since Objective-C frameworks tend to favour a
12/// design pattern using "delegates", where to hook into a piece of
13/// functionality in a class, you implement that class' delegate protocol in
14/// a custom class.
15///
16/// This macro is the declarative way of creating classes, in contrast with
17/// [`ClassBuilder`], which allows creating classes in an imperative fashion.
18/// It is highly recommended that you use this macro though, since it contains
19/// a lot of extra debug assertions and niceties that help ensure the
20/// soundness of your code.
21///
22/// The class is guaranteed to have been created and registered with the
23/// Objective-C runtime after the [`ClassType::class`] function has been
24/// called.
25///
26/// See [Apple's documentation] on defining classes for a more in-depth
27/// introduction.
28///
29/// [Apple's documentation]: https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC/DefiningClasses/DefiningClasses.html
30/// [`ClassBuilder`]: crate::runtime::ClassBuilder
31/// [`ClassType::class`]: crate::ClassType::class
32///
33///
34/// # Specification
35///
36/// This macro consists of the following parts:
37/// - The type definition, along with special attributes and it's fields.
38/// - Any number of inherent implementations.
39/// - Any number of protocol implementations.
40///
41/// With the syntax generally resembling a combination of that in
42/// [`extern_class!`] and [`extern_methods!`].
43///
44/// This macro creates an opaque struct with implementations in [a similar
45/// manner as the `extern_class!` macro][ec_spec]. Additionally, it implements
46/// the [`DefinedClass`] trait, as well as any protocols specified in the
47/// protocol implementations.
48///
49/// If the type implements [`Drop`], the macro will generate a `dealloc`
50/// method for you, which will call `drop` automatically.
51///
52/// The macro does not support generic types.
53///
54/// [`extern_class!`]: crate::extern_class
55/// [`extern_methods!`]: crate::extern_methods
56/// [ec_spec]: crate::extern_class#specification
57/// [`DefinedClass`]: crate::DefinedClass
58///
59///
60/// ## Instance variables (struct fields)
61///
62/// Fields on the struct correspond to [instance variables] on the class. This
63/// is the intended way to specify the data your class stores. Fields are not
64/// directly accessible, since the offsets needed to access them cannot be
65/// computed at compile-time. Instead, this macro generates corresponding
66/// getter functions, e.g. a field `foo: i32` generates a function
67/// `fn foo(&self) -> &i32`.
68///
69/// Additionally, you can access the semi-internal helper struct that contains
70/// all the fields with the [`Ivars`] type alias, e.g. as `Ivars::<MyClass>`.
71/// This is useful when initializing the class with [`set_ivars`].
72///
73/// It is recommended that you wrap your instance variables in [`Cell`],
74/// [`RefCell`], atomics or other similar interior mutability abstractions to
75/// allow mutating your instance variables. See [the docs on interior
76/// mutability][interior_mutability] for further details.
77///
78/// Beware that if you want to use the class' inherited initializers (such as
79/// `init`), you must override the subclass' designated initializers, and
80/// initialize your ivars properly in there.
81///
82/// [instance variables]: https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/ObjectiveC/Chapters/ocDefiningClasses.html#//apple_ref/doc/uid/TP30001163-CH12-TPXREF126
83/// [`set_ivars`]: crate::rc::Allocated::set_ivars
84/// [`Ivars`]: crate::Ivars
85/// [`Cell`]: core::cell::Cell
86/// [`RefCell`]: core::cell::RefCell
87/// [interior_mutability]: crate::topics::interior_mutability
88///
89///
90///
91/// ## Attributes
92///
93/// You can add most normal attributes to the class, including `#[cfg(...)]`,
94/// `#[allow(...)]` and doc comments.
95///
96/// Exceptions and special attributes are noted below.
97///
98///
99/// ### `#[unsafe(super(...))]` (required)
100///
101/// Same [as in `extern_class!`](crate::extern_class#unsafesuper-required).
102///
103///
104/// ### `#[thread_kind = ...]` (optional)
105///
106/// Same [as in `extern_class!`](crate::extern_class#thread_kind---optional).
107///
108///
109/// ### `#[name = "..."]` (optional)
110///
111/// Specify the runtime-name for the class. Must be unique across the entire
112/// application. This is useful if the name of a class is used elsewhere, such
113/// as when defining a delegate that needs to be named in e.g. `Info.plist`.
114///
115/// If not set, this will default to:
116/// ```ignore
117/// concat!(module_path!(), "::", $class, env!("CARGO_PKG_VERSION"));
118/// ```
119///
120/// E.g. for example `"my_crate::my_module::MyClass0.1.0"`.
121///
122/// If you're developing a library, it is recommended that you do not set
123/// this, and instead rely on the default naming, since that usually works
124/// better with users having multiple SemVer-incompatible versions of your
125/// library in the same binary.
126///
127/// If the name is auto-generated, the class will also be allowed to be used
128/// across multiple shared dynamic libraries in the same process.
129///
130///
131/// ### `#[derive(...)]`
132///
133/// This is overridden, and only works with [`PartialEq`], [`Eq`], [`Hash`]
134/// and [`Debug`].
135///
136/// The implementations delegate to the superclass' implementation, so if you
137/// want to change how they work, you should override the [`isEqual:`] and
138/// [`hash`] methods instead.
139///
140/// The `Debug` implementation currently also debug print your ivars, but that
141/// may change in the future. Prefer to override [`description`] (and
142/// potentially [`debugDescription`]) instead.
143///
144/// [`Hash`]: std::hash::Hash
145/// [`Debug`]: std::fmt::Debug
146/// [`isEqual:`]: crate::runtime::NSObjectProtocol::isEqual
147/// [`hash`]: crate::runtime::NSObjectProtocol::hash
148/// [`description`]: crate::runtime::NSObjectProtocol::description
149/// [`debugDescription`]: crate::runtime::NSObjectProtocol::debugDescription
150///
151///
152/// ### `#[cfg_attr(..., ...)]`
153///
154/// Same [as in `extern_class!`](crate::extern_class#cfg_attr-).
155///
156///
157/// ### `#[repr(...)]`
158///
159/// Same [as in `extern_class!`](crate::extern_class#repr).
160///
161///
162/// ## Inherent method definitions
163///
164/// Within the `impl` block you can define two types of functions;
165/// ["associated functions"] and ["methods"]. These are then mapped to the
166/// Objective-C equivalents "class methods" and "instance methods". In
167/// particular, if you use `self` or the special name `this` (or `_this`),
168/// your method will be registered as an instance method, and if you don't it
169/// will be registered as a class method.
170///
171/// On instance methods, you can freely choose between different types of
172/// receivers, e.g. `&self`, `self: *const Self`, `this: *const Self`, and so
173/// on. Note that using `&mut self` is not possible, if you need mutation of
174/// your class' instance variables, consider using [`Cell`] or similar
175/// instead.
176///
177/// The desired selector can be specified using the
178/// `#[unsafe(method(my:selector:))]` or `#[unsafe(method_id(my:selector:))]`
179/// attributes, similar to the [`extern_methods!`] macro.
180///
181/// If the `#[unsafe(method_id(...))]` attribute is used, the return type must
182/// be `Option<Retained<T>>` or `Retained<T>`. Additionally, if the selector
183/// is in the "init"-family, the `self`/`this` parameter must be
184/// `Allocated<Self>`.
185///
186/// Putting other attributes on the method such as `cfg`, `allow`, `doc`,
187/// `deprecated` and so on is supported. However, note that `cfg_attr` may not
188/// work correctly, due to implementation difficulty - if you have a concrete
189/// use-case, please [open an issue], then we can discuss it.
190///
191/// A transformation step is performed on the functions (to make them have the
192/// correct ABI) and hence they shouldn't really be called manually. (You
193/// can't mark them as `pub` for the same reason). Instead, use the
194/// [`extern_methods!`] macro to create a Rust interface to the methods.
195///
196/// If the parameter or return type is [`bool`], a conversion is performed to
197/// make it behave similarly to the Objective-C `BOOL`. Use [`runtime::Bool`]
198/// if you want to control this manually.
199///
200/// Note that `&mut Retained<_>` and other such out parameters are not yet
201/// supported, and may generate a panic at runtime.
202///
203/// ["associated functions"]: https://doc.rust-lang.org/reference/items/associated-items.html#methods
204/// ["methods"]: https://doc.rust-lang.org/reference/items/associated-items.html#methods
205/// [`Cell`]: core::cell::Cell
206/// [open an issue]: https://github.com/madsmtm/objc2/issues/new
207/// [`msg_send!`]: crate::msg_send
208/// [`runtime::Bool`]: crate::runtime::Bool
209///
210///
211/// ## Protocol implementations
212///
213/// You can specify protocols that the class should implement, along with any
214/// required/optional methods for said protocols.
215///
216/// The protocol must have been previously defined with [`extern_protocol!`].
217///
218/// The methods work exactly as normal, they're only put "under" the protocol
219/// definition to make things easier to read.
220///
221/// Putting attributes on the `impl` item such as `cfg`, `allow`, `doc`,
222/// `deprecated` and so on is supported.
223///
224/// [`extern_protocol!`]: crate::extern_protocol
225///
226///
227/// # Panics
228///
229/// The implemented `ClassType::class` method may panic in a few cases, such
230/// as if:
231/// - A class with the specified name already exists.
232/// - Debug assertions are enabled, and an overridden method's signature is not
233///   equal to the one on the superclass.
234/// - Debug assertions are enabled, and the protocol's required methods are not
235///   implemented.
236///
237/// And possibly more similar cases in the future.
238///
239///
240/// # Safety
241///
242/// Using this macro requires writing a lot of `unsafe` markers:
243///
244/// When writing `#[unsafe(super(...))]`, you must ensure that:
245/// - Any invariants that the superclass [`ClassType::Super`] may have must be
246///   upheld.
247/// - If your type implements `Drop`, the implementation must abide by the
248///   following rules:
249///   - It must not call any overridden methods.
250///   - It must not `retain` the object past the lifetime of the drop.
251///   - It must not `retain` in the same scope that `&mut self` is active.
252///   - TODO: And probably a few more. [Open an issue] if you would like
253///     guidance on whether your implementation is correct.
254///
255/// `#[unsafe(method(...))]` asserts that the types match those that are
256/// expected when the method is invoked from Objective-C. Note that unlike
257/// with [`extern_methods!`], there are no safe-guards here; you can write
258/// `i8`, but if Objective-C thinks it's an `u32`, it will cause UB when
259/// called!
260///
261/// `unsafe impl P for T { ... }` requires that all required methods of the
262/// specified protocol is implemented, and that any extra requirements
263/// (implicit or explicit) that the protocol has are upheld.
264///
265/// [`ClassType::Super`]: crate::ClassType::Super
266/// [Open an issue]: https://github.com/madsmtm/objc2/issues/new
267///
268///
269/// # Thread safety
270///
271/// The Objective-C runtime is thread-safe, so any classes you create yourself
272/// will automatically be [`Send`] and [`Sync`] (via auto traits) provided
273/// that:
274/// 1. The superclass is thread-safe, or is [`NSObject`].
275/// 2. The ivars are thread-safe.
276/// 3. The thread kind is not [`MainThreadOnly`].
277///
278/// Note though that in many cases, [the frameworks] you will be interacting
279/// with will not be thread-safe, and so in many cases it will make sense to
280/// [use interior mutability] in your custom classes.
281///
282/// [`NSObject`]: crate::runtime::NSObject
283/// [`MainThreadOnly`]: crate::MainThreadOnly
284/// [the frameworks]: crate::topics::frameworks_list
285/// [use interior mutability]: crate::topics::interior_mutability
286///
287///
288/// # Examples
289///
290/// Define a class `MyCustomObject` that inherits `NSObject`, has a few
291/// instance variables and methods, and implements the `NSCopying` protocol.
292///
293/// ```
294/// use std::ffi::c_int;
295///
296/// # use objc2::runtime::{NSObject, NSObjectProtocol, NSZone};
297/// # objc2::extern_protocol!(
298/// #     unsafe trait NSCopying {
299/// #         #[unsafe(method(copy))]
300/// #         fn copy(&self) -> Retained<Self>;
301/// #         #[unsafe(method(copyWithZone:))]
302/// #         unsafe fn copyWithZone(&self, zone: *mut NSZone) -> Retained<Self>;
303/// #     }
304/// # );
305/// # #[cfg(requires_foundation)]
306/// use objc2_foundation::{CopyingHelper, NSCopying, NSObject, NSObjectProtocol, NSZone};
307/// use objc2::rc::{Allocated, Retained};
308/// use objc2::{
309///     define_class, extern_methods, extern_protocol, msg_send, AnyThread,
310///     ClassType, Ivars, ProtocolType,
311/// };
312///
313/// define_class!(
314///     // SAFETY:
315///     // - The superclass NSObject does not have any subclassing requirements.
316///     // - `MyCustomObject` does not implement `Drop`.
317///     #[unsafe(super(NSObject))]
318///
319///     // If we were implementing delegate methods like `NSApplicationDelegate`,
320///     // we would specify the object to only be usable on the main thread:
321///     // #[thread_kind = MainThreadOnly]
322///
323///     // If we needed to refer to the class from elsewhere, we'd give it a
324///     // name here explicitly.
325///     // #[name = "MyCustomObject"]
326///
327///     struct MyCustomObject {
328///         // Specify the instance variables this class has.
329///         foo: u8,
330///         bar: c_int,
331///         object: Retained<NSObject>,
332///     }
333///
334///     impl MyCustomObject {
335///         // Access instance variable, and expose it to Objective-C.
336///         #[unsafe(method(foo))]
337///         fn __get_foo(&self) -> u8 {
338///             *self.foo()
339///         }
340///
341///         #[unsafe(method_id(object))]
342///         fn __get_object(&self) -> Retained<NSObject> {
343///             self.object().clone()
344///         }
345///
346///         #[unsafe(method(myClassMethod))]
347///         fn __my_class_method() -> bool {
348///             true
349///         }
350///     }
351///
352///     unsafe impl NSObjectProtocol for MyCustomObject {}
353///
354///     unsafe impl NSCopying for MyCustomObject {
355///         #[unsafe(method_id(copyWithZone:))]
356///         fn copyWithZone(&self, _zone: *const NSZone) -> Retained<Self> {
357///             let new = Self::alloc().set_ivars(Ivars::<Self> {
358///                 foo: *self.foo(),
359///                 bar: *self.bar(),
360///                 object: self.object().clone(),
361///             });
362///             unsafe { msg_send![super(new), init] }
363///         }
364///
365///         // If we have tried to add other methods here, or had forgotten
366///         // to implement the method, we would have gotten an error.
367///     }
368/// );
369///
370/// // Specially required for `NSCopying`, but otherwise not needed.
371/// # #[cfg(requires_foundation)]
372/// unsafe impl CopyingHelper for MyCustomObject {
373///     type Result = Self;
374/// }
375///
376/// // Add creation method.
377/// impl MyCustomObject {
378///     fn new(foo: u8) -> Retained<Self> {
379///         // Initialize instance variables.
380///         let this = Self::alloc().set_ivars(Ivars::<Self> {
381///             foo,
382///             bar: 42,
383///             object: NSObject::new(),
384///         });
385///         // Call `NSObject`'s `init` method.
386///         unsafe { msg_send![super(this), init] }
387///     }
388/// }
389///
390/// // Make an interface to the methods we defined.
391/// impl MyCustomObject {
392///     extern_methods!(
393///         #[unsafe(method(foo))]
394///         pub fn get_foo(&self) -> u8;
395///
396///         #[unsafe(method(object))]
397///         pub fn get_object(&self) -> Retained<NSObject>;
398///
399///         #[unsafe(method(myClassMethod))]
400///         pub fn my_class_method() -> bool;
401///     );
402/// }
403///
404/// # // Intentionally use `fn main` for clarity
405/// fn main() {
406///     let obj = MyCustomObject::new(3);
407///     assert_eq!(*obj.foo(), 3);
408///     assert_eq!(*obj.bar(), 42);
409///     assert!(obj.object().isKindOfClass(NSObject::class()));
410///
411///     let obj = obj.copy();
412///
413///     assert_eq!(obj.get_foo(), 3);
414///     assert!(obj.get_object().isKindOfClass(NSObject::class()));
415///
416///     assert!(MyCustomObject::my_class_method());
417/// }
418/// ```
419///
420/// Approximately equivalent to the following ARC-enabled Objective-C code.
421///
422/// ```text
423/// #import <Foundation/Foundation.h>
424///
425/// @interface MyCustomObject: NSObject <NSCopying>
426/// - (instancetype)initWithFoo:(uint8_t)foo;
427/// - (uint8_t)foo;
428/// - (NSObject*)object;
429/// + (BOOL)myClassMethod;
430/// @end
431///
432///
433/// @implementation MyCustomObject {
434///     // Instance variables
435///     uint8_t foo;
436///     int bar;
437///     NSObject* _Nonnull object;
438/// }
439///
440/// - (instancetype)initWithFoo:(uint8_t)foo_arg {
441///     self = [super init];
442///     if (self) {
443///         self->foo = foo_arg;
444///         self->bar = 42;
445///         self->object = [NSObject new];
446///     }
447///     return self;
448/// }
449///
450/// - (uint8_t)foo {
451///     return self->foo;
452/// }
453///
454/// - (NSObject*)object {
455///     return self->object;
456/// }
457///
458/// + (BOOL)myClassMethod {
459///     return YES;
460/// }
461///
462/// // NSCopying
463///
464/// - (id)copyWithZone:(NSZone *)_zone {
465///     MyCustomObject* new = [[MyCustomObject alloc] init];
466///     new->foo = self->foo;
467///     new->bar = self->bar;
468///     new->obj = self->obj;
469///     return new;
470/// }
471///
472/// @end
473/// ```

476#[macro_export]
477macro_rules! define_class {
478    {
479        // The following special attributes are supported:
480        // - #[unsafe(super($($superclasses:path),*))]
481        // - #[unsafe(super = $superclass:path)]
482        // - #[thread_kind = $thread_kind:path]
483        // - #[name = $name:literal]
484        $(#[$($attrs:tt)*])*
485        $v:vis struct $class:ident {
486            // Instance variables (written as struct fields).
487            $($ivars:tt)*
488        }
489
490        // unsafe impl Protocol for $class { ... }
491        // impl $class { ... }
492        $($impls:tt)*
493    } => {
494        // Struct and various impls.
495        $crate::__extract_attributes! {
496            ($(#[$($attrs)*])*)
497
498            ($crate::__define_class_inner)
499            ($v)
500            ($class)
501            ($($ivars)*)
502            // We duplicate the impls here, since we need them to create a
503            // thunk that is inserted into the class' method table.
504            ($($impls)*)
505        }
506
507        // Methods.
508        $crate::__define_class_output_impls! {
509            // TODO: Add $class here and verify that the same item is being
510            // implemented everywhere.
511            $($impls)*
512        }
513    };
514
515    {
516        // Same as above, but without any instance variables.
517        $(#[$($attrs:tt)*])*
518        $v:vis struct $class:ident;
519
520        $($impls:tt)*
521    } => {
522        $crate::define_class! {
523            $(#[$($attrs)*])*
524            $v struct $class {
525                // No ivars
526            }
527
528            $($impls)*
529        }
530    };
531}

563#[doc(hidden)]
564#[macro_export]
565macro_rules! __define_class_inner {
566    (
567        ($v:vis)
568        ($class:ident)
569        ($($ivars:tt)*)
570        ($($impls:tt)*)
571
572        ($($safety:tt $superclass:path $(, $superclasses:path)* $(,)?)?)
573        ($($($thread_kind:tt)+)?)
574        ($($name:tt)*)
575        ($($derives:tt)*)
576        ($($attr_struct:tt)*)
577        ($($attr_impl:tt)*)
578    ) => {
579        // Ensure that the type has the same layout as the superclass.
580        // #[repr(transparent)] doesn't work because the superclass is a ZST.
581        #[repr(C)]
582        $($attr_struct)*
583        $v struct $class {
584            // Superclasses are deallocated by calling `[super dealloc]`.
585            //
586            // Auto traits are taken from `__SubclassingType` (which is
587            // usually the super class).
588            __superclass: $crate::__macros::ManuallyDrop<$crate::__fallback_if_not_set! {
589                ($(<$superclass as $crate::ClassType>::__SubclassingType)?)
590                // For better diagnostics, see also __extern_class_inner!
591                ($crate::runtime::NSObject)
592            }>,
593            __phantom: $crate::__macros::PhantomData<(
594                // Include ivars for auto traits.
595                <Self as $crate::DefinedClass>::Ivars,
596                // Translate thread kind to appropriate auto traits.
597                $crate::__macros::ThreadKindAutoTraits<<Self as $crate::ClassType>::ThreadKind>,
598            )>,
599        }
600
601        $crate::__extern_class_impl_traits! {
602            ($($attr_impl)*)
603            (unsafe impl)
604            ($class)
605            ($($superclass, $($superclasses,)*)? $crate::runtime::AnyObject)
606        }
607
608        $crate::__define_class_derives! {
609            ($($attr_impl)*)
610            ($class)
611            ($($ivars)*)
612            ($($derives)*)
613        }
614
615        // Anonymous block to hide the shared statics
616        $($attr_impl)*
617        const _: () = {
618            // Add #[export_name] here to ensure that all class names are
619            // unique, at least within the resulting linker invocation (though
620            // not across dylibs). The Objective-C runtime will ensure this as
621            // well, but this will allow ensuring it at link time.
622            #[export_name = $crate::__macros::concat!(
623                "__CLASS_",
624                $crate::__define_class_name!($class, $($name)*),
625            )]
626            static __OBJC2_CLASS: $crate::__macros::SyncUnsafeCell<
627                $crate::__macros::MaybeUninit<&'static $crate::runtime::AnyClass>
628            > = $crate::__macros::SyncUnsafeCell::new($crate::__macros::MaybeUninit::uninit());
629
630            #[export_name = $crate::__macros::concat!(
631                "__IVAR_OFFSET_",
632                $crate::__define_class_name!($class, $($name)*),
633            )]
634            static __OBJC2_IVAR_OFFSET: $crate::__macros::SyncUnsafeCell<
635                $crate::__macros::MaybeUninit<$crate::__macros::isize>
636            > = $crate::__macros::SyncUnsafeCell::new($crate::__macros::MaybeUninit::uninit());
637
638            #[export_name = $crate::__macros::concat!(
639                "__DROP_FLAG_OFFSET_",
640                $crate::__define_class_name!($class, $($name)*),
641            )]
642            static __OBJC2_DROP_FLAG_OFFSET: $crate::__macros::SyncUnsafeCell<
643                $crate::__macros::MaybeUninit<$crate::__macros::isize>
644            > = $crate::__macros::SyncUnsafeCell::new($crate::__macros::MaybeUninit::uninit());
645
646            // Creation
647            unsafe impl $crate::ClassType for $class {
648                type Super = $crate::__fallback_if_not_set! {
649                    ($($superclass)?)
650                    // For better diagnostics, see also __extern_class_inner!
651                    ($crate::runtime::NSObject)
652                };
653
654                type ThreadKind = $crate::__fallback_if_not_set! {
655                    ($(dyn ($($thread_kind)+))?)
656                    // Default to the super class' thread kind
657                    (<<Self as $crate::ClassType>::Super as $crate::ClassType>::ThreadKind)
658                };
659
660                const NAME: &'static $crate::__macros::str = $crate::__define_class_name!($class, $($name)*);
661
662                fn class() -> &'static $crate::runtime::AnyClass {
663                    let _ = <Self as $crate::__macros::ValidThreadKind<Self::ThreadKind>>::check;
664                    let _ = <Self as $crate::__macros::MainThreadOnlyDoesNotImplSendSync<_>>::check;
665
666                    const C_NAME: &'static $crate::__macros::CStr = $crate::__macros::class_c_name(
667                        $crate::__macros::concat!($crate::__define_class_name!($class, $($name)*), "\0")
668                    );
669
670                    // TODO: Use `std::sync::OnceLock`
671                    #[export_name = $crate::__macros::concat!(
672                        "__REGISTER_CLASS_",
673                        $crate::__define_class_name!($class, $($name)*),
674                    )]
675                    static REGISTER_CLASS: $crate::__macros::Once = $crate::__macros::Once::new();
676
677                    REGISTER_CLASS.call_once(|| {
678                        let (__objc2_cls, __objc2_ivar_offset, __objc2_drop_flag_offset) = $crate::__macros::define_class::<Self>(
679                            C_NAME,
680                            $crate::__define_class_name_is_auto_generated!($($name)*),
681                            |mut __objc2_builder| {
682                                // Implement protocols and methods
683                                $crate::__define_class_register_impls! {
684                                    (__objc2_builder)
685                                    $($impls)*
686                                }
687                            }
688                        );
689
690                        // SAFETY: Modification is ensured by `Once` to happen
691                        // before any access to the variables.
692                        unsafe {
693                            __OBJC2_CLASS.get().write($crate::__macros::MaybeUninit::new(__objc2_cls));
694                            if <Self as $crate::__macros::DefinedIvarsHelper>::HAS_IVARS {
695                                __OBJC2_IVAR_OFFSET.get().write($crate::__macros::MaybeUninit::new(__objc2_ivar_offset));
696                            }
697                            if <Self as $crate::__macros::DefinedIvarsHelper>::HAS_DROP_FLAG {
698                                __OBJC2_DROP_FLAG_OFFSET.get().write($crate::__macros::MaybeUninit::new(__objc2_drop_flag_offset));
699                            }
700                        }
701                    });
702
703                    // SAFETY: We just registered the class, so is now available
704                    unsafe { __OBJC2_CLASS.get().read().assume_init() }
705                }
706
707                #[inline]
708                fn as_super(&self) -> &Self::Super {
709                    &*self.__superclass
710                }
711
712                const __INNER: () = ();
713
714                type __SubclassingType = Self;
715            }
716
717            // Pub because everyone that see `$class` can access this via
718            // `<$class as DefinedClass>::Ivars`.
719            #[allow(unreachable_pub)]
720            pub struct __Objc2IvarsContainer {
721                // These are intentionally not parsed when inserted here, to
722                // allow rust-analyzer to better work with incomplete fields.
723                $($ivars)*
724            }
725
726            $crate::__define_class_ivar_accessors!(($class) $($ivars)*);
727
728            impl $crate::DefinedClass for $class {
729                // NOTE: We could have used `()` here when there are no ivars,
730                // but it's more consistent to always use the container.
731                type Ivars = __Objc2IvarsContainer;
732
733                #[inline]
734                fn __ivars_offset() -> $crate::__macros::isize {
735                    // Only access ivar offset if we have an ivar.
736                    //
737                    // This makes the offset not be included in the final
738                    // executable if it's not needed.
739                    if <Self as $crate::__macros::DefinedIvarsHelper>::HAS_IVARS {
740                        // SAFETY: Accessing the offset is guaranteed to only be
741                        // done after the class has been initialized.
742                        unsafe { __OBJC2_IVAR_OFFSET.get().read().assume_init() }
743                    } else {
744                        // Fall back to an offset of zero.
745                        //
746                        // This is fine, since any reads here will only be via zero-sized
747                        // ivars, where the actual pointer doesn't matter.
748                        0
749                    }
750                }
751
752                #[inline]
753                fn __drop_flag_offset() -> $crate::__macros::isize {
754                    if <Self as $crate::__macros::DefinedIvarsHelper>::HAS_DROP_FLAG {
755                        // SAFETY: Same as above.
756                        unsafe { __OBJC2_DROP_FLAG_OFFSET.get().read().assume_init() }
757                    } else {
758                        // Fall back to an offset of zero.
759                        //
760                        // This is fine, since the drop flag is never actually used in the
761                        // cases where it was not added.
762                        0
763                    }
764                }
765
766                // SAFETY: The offsets are implemented correctly
767                const __UNSAFE_OFFSETS_CORRECT: () = ();
768            }
769        };
770
771        // SAFETY: This macro only allows non-generic classes and non-generic
772        // classes are always valid downcast targets.
773        $($attr_impl)*
774        unsafe impl $crate::DowncastTarget for $class {}
775
776        $($attr_impl)*
777        $crate::__extern_class_check_super_unsafe!($($safety $superclass)?);
778    };
779}
